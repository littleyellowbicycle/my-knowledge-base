## 三、Obsidian 端配置与插件指南

Obsidian 在本系统中承担**“交互前端 + 渲染器 + 自动同步器”**的角色。

### 1. 基础配置
* **打开仓库**：选择“打开文件夹作为仓库”，选中 `my_kb/` 文件夹。
* **忽略文件**：进入 `设置 -> 文件与链接 -> 忽略文件`，添加 `raw` 和 `index`，防止 Obsidian 索引冗余文件导致卡顿。

### 2. 核心插件清单 (社区插件市场安装)

#### A. Shell Commands (核心触发器)
* **作用**：在 Obsidian 内部通过快捷键执行后端脚本，告别命令行。
* **配置**：
  * 新建命令：`python main.py ingest "{{clipboard}}"` (抓取剪贴板链接)
  * 新建命令：`python main.py qa "{{selection}}"` (对选中文字提问)
  * 在 Obsidian 快捷键设置中为上述命令分配全局快捷键。

#### B. Obsidian Git (云端同步器)
* **作用**：自动将本地知识库同步到 GitHub，实现多端访问和版本控制。
* **配置**：
  * **Auto backup & push**：设置为 `Every 10 minutes`。
  * **Commit message**：设置为 `chore(auto-sync): update knowledge base at {{date}}`。
  * **Pull on startup**：开启（确保手机端拉取最新加工笔记）。

### 3. 进阶体验插件 (可选)

#### C. Templater (或 QuickAdd)
* **作用**：配合 `inbox/` 目录，一键创建“闪念笔记”或“待抓取链接笔记”。
* **配置**：创建空白模板，配置快捷键一键在 `my_kb/inbox/` 下新建笔记。

#### D. Smart Connections 或 Copilot for Obsidian (侧边栏对话)
* **作用**：提供类似 ChatGPT 的侧边栏聊天体验。
* **配置**：将插件的 API Base URL 修改为本地 FastAPI 地址 `http://localhost:8000`，使回答 100% 基于本地知识库。

## 四、V2.1 终极工作流体验

配置完毕后，你的日常操作将完全融入 Obsidian，实现真正的零摩擦：

1. **灵感闪现**：`Ctrl+N` 在 `inbox/` 新建笔记 -> 写下想法 -> `Ctrl+S` -> (几秒后 Obsidian 文件树自动出现加工好的结构化笔记，双链自动生成)。
2. **网页收集**：浏览器复制链接 -> Obsidian `inbox/` 新建笔记粘贴链接 -> `Ctrl+S` -> (几秒后自动生成带核心结论的阅读笔记)。
3. **知识问答**：在笔记里写下问题 -> 选中文字按快捷键 -> (下方自动追加基于本地笔记的精准回答及 `[[]]` 来源链接)。
4. **无缝同步**：你只管写和读，`Obsidian Git` 每隔 10 分钟默默把所有新增双链和综述页推送到 GitHub，手机端随时可查。
