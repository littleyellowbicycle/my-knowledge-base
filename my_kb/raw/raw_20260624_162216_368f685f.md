## 所用开源项目：

先罗列几个开源项目，大部分应该都是耳熟能详的：

1、alphalens：一套用于分析 alpha 因子的通用工具，出品于大名鼎鼎的Quantopian 公司。

2、bt：一个小巧简单的基于Python的量化策略*回测*工具，不是backtrader的简称，比backtrader简单很多。

3、alphas：一个简单的alpha101和alpha191实现和测试。

另外，还有免费或半免费的数据源：

1、baostock：证券宝www.*baostock*.com是一个免费、开源的证券数据平台(无需注册)。 提供大量准确、完整的证券历史行情数据、上市公司财务数据等。

2、akshare：也是基于 Python 的开源金融数据接口库。后台实际上是对东方财富网的页面数据抓取。

本框架的主体逻辑是基于alphas项目的，但对回测框架进行了替换，数据源进行了调整，另外增加了批量化的处理和界面的操作。

## 使用步骤：

A、基础数据准备：

1. 运行datas.py下载股票数据（先不用太在意前复权还是后复权，因为很多原始数据不准确，这个是个大问题，后面会讨论）
2. 运行alphas101.py或alphas191.py计算因子
3. 在1.research.ipynb里分析因子（这里需要对原始的alphalens进行简单修改以适配新的版本库，我的框架基本上也不用这个）
4. 在2.backtrader.ipynb回测（这里原始使用的是backtrader回测框架，但backtrader添加基准信息一直不成功，而且批量多因子全市场回测不好做，所以我改成了bt进行回测，使用2.1.bt.ipynb）
5. 在factorycompare.ipynb计算历年所有因子的收益并进行百分比排名（这个我没用）

### 5、bt自定义策略类

```
class SelectTrade(bt.Algo):

    def __init__(self, signal, alpha:str):
        self.signal = signal
        self.alpha = alpha

    def __call__(self, target):
        # get signal on target.now
        if target.now in self.signal.index:
            print(f'{self.alpha} set selected {target.now}')
            sig = self.signal.loc[target.now]

            # get indices where true as list
            selected = list(sig.index[sig])

            # save in temp - this will be used by the weighing algo
            target.temp['selected'] = selected
        else:
            target.temp['selected'] = []
        # return True because we want to keep on moving down the stack
        return True
```

bt能够同时对多个标的或策略进行回测，并且执行效率很高，这得益于bt的优秀设计，这是我使用bt的主要原因，后面计划详细研究介绍下bt。

bt_list = []
for i in range(alpha_all):

## 个人对量化系统的理解

本人接触量化最早是国内的聚宽平台，当时就知道写一个会被系统回调的策略函数，因为不能实盘，也没什么感受。后来接触到backtrader，研究了很长一段时间，但还是不得要领，一直想把它的实盘接口在A股上实现，但一直没完成，而且源代码嵌套继承非常多，基本没法调试，熟悉它的数据结构的各种概念就花了挺长时间，后来也就基本上不了了之了。

知道最近接触到一个叫wondertrader的C++实现的量化平台，我可能才终于对量化交易系统有了比较清晰的认识。可能由于我本身就是C++程序员出身的缘故吧，对C++项目有着天生的亲切感，熟悉的架构、熟悉的代码组织、熟悉事件框架，我基本上花了几天时间（也是大A跌得最猛的那几天吧），就对wondertrader的框架有了大概的了解，也确定了我个人短时间之内不需要wondertrader这样复杂高性能的交易系统。我暂时需要的，只是对策略/因子（更确切的讲，只是因子）能够整体地同时进行回测，从而发现有效因子，发现有效因子后能够根据实盘数据先进行模拟交易，然后再实盘交易。

量化回测系统，如果从我的程序员架构角度讲，其实就是一个遍历股票K线数据的for循环，每次for循环执行一定的计算，大部分的开源框架提供的能够自定义的策略函数，就是for循环内需要回调的主要函数。而量化交易系统，就是用实盘数据获取过程代替了for循环，然后策略函数内对接交易接口进行下单操作。当然实际的细节处理会复杂很多，但大的逻辑就是这样简单。更简单的操作，甚至可以根本不需要这些什么框架，直接使用数据库的函数就可以，然后对函数执行定时任务，定时根据历史K线数据（昨日+历史，或更多其它市场数据）计算取得需要买入卖出信号，并生成结果，交易程序（如miniqmt）扫描数据库，对交易信号实时进行处理，本质上就如同我们的互联网公司每日晨会review数据报表，然后决策当日的运营投放等计划等几无差别。当然这是相对较简单的，高频的系统会有点不一样，但基本的概念是一样的。
