## 框架介绍

本质上，这是一套基于贝叶斯推理的 A 股基本面研究框架，包含两个互补工具：

1. **Serenity Alpha** — 将市场新闻转化为可验证的 Alpha 假设
2. **贝叶斯内在增长估值** — 用概率更新评估公司真实增长速度 vs 市场定价

### 设计理念

传统投资分析容易陷入”利好/利空”的二元判断。Serenity 的核心思想是：

> **不要把新闻简单归类为利好或利空。把每个数据点转化为对未来 3-5 年收入 CAGR 的概率更新。**

### 框架组成

### 一、Serenity Alpha（新闻 → Alpha 假设）

将新闻/事件/公告转化为可验证的投资假设，核心流程：

```
新闻 → 已观察到的需求变化 → 财务传导路径 → 小市值弹性 → 验证路径
```

输出一份结构化的 Alpha 报告，包含：

* 结论先行（点名最值得验证的公司）
* 需求变化分析（区分”叙事”vs”已发生的需求变化”）
* 财务翻译（需求如何进入利润表）
* 小市值高弹性标的筛选
* 市场误分类测试
* 验证指标（1-4 季度内可验证）
* Alpha 强度评分

### 二、贝叶斯内在增长估值

用概率思维评估公司真实增长速度，与市场定价对比：

* **6 个增长假设**：从 H0（衰退 <0%）到 H5（平台级扩张 >50%）
* **先验 → 新信息 → 后验**：每条信息都更新各假设的概率
* **加权内在增长速度**：根据后验概率估算 CAGR
* **市场隐含增长速度**：反推当前股价/市值已隐含的增长率
* **估值状态判断**：低估 / 合理 / 高估但可交易 / 泡沫化

### 触发场景

| 场景 | 使用方法 |
| --- | --- |
| 分享一条新闻/公告 | Serenity Alpha — 分析是否产生 Alpha 机会 |
| 问某公司值不值得买 | 贝叶斯估值 — 内在增长 vs 市场定价 |
| 问新闻对持仓的影响 | Alpha + 贝叶斯结合 |
| 连续分析多只同主题股票 | 独立分析 + 横截面比较 |

### 数据源

框架整合了以下数据源（具体命令见 `references/data-gathering-workflow.md`）：

* **实时行情** → Sina Finance API（curl, 无浏览器）
* **估值/行业排名** → 东方财富 push2 API + 行情页
* **财务时间序列** → 同花顺 F10 财务页
* **公告/减持/龙虎榜** → 东方财富公告页
* **新闻** → 新浪财经首页
* **K 线分析** → Sina 30分钟/5分钟 API
* **实时资金流向** → 东方财富行情页底部

### 注意事项

* 所有输出均为研究假设，不构成投资建议
* 重视 **内部人减持信号** — 股票暴涨期间控股股东同步减持是最强烈的警示
* 重视 **股东户数变化** — 筹码分散是中期方向的重要信号
* 数据交叉验证：Sina 实时价 vs 东方财富估值，避免单一来源错误
