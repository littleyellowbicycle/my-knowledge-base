# Example 域名：文档示例专用保留域名

## 核心结论
> example.com 属于 [[IANA]] 专门保留的示例域名，仅供文档和示例场景使用，严禁用于实际生产或运营环境。其作用是为技术文档、教程和测试提供统一、安全的占位符，避免开发者误用真实域名。

## 详细内容
## 背景与定位

[[example.com]]（以及 [[example.org]]、[[example.net]]）是 [[IANA]]（互联网号码分配机构）根据 [[RFC 2606]] 和 [[RFC 6761]] 专门保留的[[二级域名]]。这些域名不指向任何真实运营的服务器，因此天然适合作为[[占位符]]使用。

## 使用规范

根据 IANA 的官方说明，example.com 域名仅用于文档示例，使用时无需获得任何授权或许可。但同时明确要求：

- 不得在生产、运营或任何实际业务场景中使用
- 适合作为 URL、邮箱地址、API 调用等场景的示例值
- 在编写技术文档、教程、单元测试时，可放心引用

## 实际应用价值

使用保留的 example 域名可以有效避免[[域名冲突]]和[[数据泄露]]风险。例如在写一段 API 调用代码时，使用 `https://example.com/api/users` 这样的 URL，比随意编造一个真实域名更规范、更安全，也便于读者理解示例意图。

## 相关资源

如需了解更多保留域名规则，可参考 [[IANA]] 官方域名根数据库，或查阅 [[RFC 2606]] 中关于测试域名和示例域名的详细规范。
