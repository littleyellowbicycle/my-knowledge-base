# Serenity 选股框架：基于贝叶斯推理的 A 股基本面研究方法论

## 核心结论
> Serenity 是一套以贝叶斯推理为核心、面向 A 股小市值标的的基本面研究框架，由 Serenity Alpha（新闻→Alpha 假设）与贝叶斯内在增长估值两个互补工具组成。其核心思想是把每一条市场信息转化为对未来 3-5 年收入 CAGR 的概率更新，从而绕开利好/利空的二元判断陷阱。框架以 AI Agent（如 Hermes Agent / Openclaw）为执行载体，整合 Sina、东方财富、同花顺等数据源，已完成一周实盘并跑赢大盘。

## 核心结论

Serenity 是一套以[[贝叶斯推理]]为核心、面向 A 股小市值标的的基本面研究框架，由 Serenity Alpha（新闻→Alpha 假设）与贝叶斯内在增长估值两个互补工具组成。其核心思想是把每一条市场信息转化为对未来 3-5 年收入 [[CAGR]] 的概率更新，从而绕开利好/利空的二元判断陷阱。框架以 AI Agent（如 [[Hermes Agent]] / Openclaw）为执行载体，整合 Sina、东方财富、同花顺等数据源，已完成一周实盘并跑赢大盘。

## 设计理念

传统投资分析容易陷入非黑即白的二元判断，而 Serenity 的关键转变在于：

> 不把新闻简单归类为利好或利空，而是把每个数据点转化为对未来 3-5 年收入 CAGR 的概率更新。

这一思路把[[基本面分析]]从主观叙事转向可量化的概率分布，使后续的估值与仓位决策建立在显式假设之上。

## 框架组成

### 一、Serenity Alpha：新闻 → Alpha 假设

将新闻、事件或公告转化为可验证的投资假设，核心流程为：

```
新闻 → 已观察到的需求变化 → 财务传导路径 → 小市值弹性 → 验证路径
```

输出是一份结构化的 Alpha 报告，包含：

- 结论先行：点名最值得验证的公司
- 需求变化分析：区分叙事与已发生的需求变化
- 财务翻译：需求如何进入利润表
- [[小市值弹性]]标的筛选
- 市场误分类测试
- 验证指标：1-4 个季度内可观察
- Alpha 强度评分

### 二、贝叶斯内在增长估值

用概率思维评估公司真实增长速度，并与市场定价对比：

- 6 个增长假设：从 H0（衰退 <0%）到 H5（平台级扩张 >50%）
- 先验 → 新信息 → 后验：每条信息都更新各假设的概率
- 加权内在增长速度：根据后验概率估算 CAGR
- 市场隐含增长速度：反推当前股价/市值已隐含的增长率
- 估值状态判断：低估 / 合理 / 高估但可交易 / 泡沫化

## 数据源整合

框架通过命令行直接调用以下 API，绕开浏览器限制：

- 实时行情 → Sina Finance API（curl 调用）
- 估值/行业排名 → 东方财富 push2 API + 行情页
- 财务时间序列 → 同花顺 F10 财务页
- 公告/减持/龙虎榜 → 东方财富公告页
- 新闻 → 新浪财经首页
- K 线分析 → Sina 30 分钟/5 分钟 API
- 实时资金流向 → 东方财富行情页底部

## 风险提示与注意事项

- 所有输出均为研究假设，不构成投资建议
- 重视[[内部人减持]]信号：股票暴涨期间控股股东同步减持是最强烈的警示
- 重视[[股东户数]]变化：筹码分散是中期方向的重要信号
- 数据交叉验证：Sina 实时价 vs 东方财富估值，避免单一来源错误
